# /grid
Restructure the target's layout — establish a clear grid, improve content flow, and create intentional spatial relationships between elements.

## When to use
- When a page feels disorganized or random
- When elements don't align to a consistent grid
- When content flow doesn't match reading patterns

## Steps
1. **Layout audit** — Identify the current layout structure. Is there a grid? Are elements aligned? Is the layout responsive?
2. **Grid system** — Establish or reinforce a grid: column count, gutter width, margin. Use the design system's grid utilities if available.
3. **Content zones** — Define clear content zones: header, navigation, main content, sidebar, footer. Ensure each zone has a clear purpose.
4. **Flow** — Arrange content to match natural reading patterns (F-pattern for text-heavy, Z-pattern for landing pages). Place primary actions in high-attention areas.
5. **Responsive** — Ensure the layout adapts across breakpoints. Stack columns on mobile, expand on desktop. Test at common breakpoints.
6. **White space** — Use white space intentionally: generous padding around primary content, tighter spacing in dense data views. White space is not wasted space.

## Output
- Restructured layout code
- Grid specification (columns, gutters, breakpoints)
- Notes on responsive behavior
