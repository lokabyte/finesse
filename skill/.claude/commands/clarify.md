# /clarify
Make the target clearer — simplify information architecture, improve labeling, reduce cognitive load, and make the user's path obvious.

## When to use
- When users report confusion or ask "what does this do?"
- When a page has too much information competing for attention
- When labels, headings, or CTAs are ambiguous

## Steps
1. **Information audit** — Map every piece of information on the page. What's essential? What's secondary? What's noise?
2. **Label review** — Check all labels, headings, button text, and placeholder text. Are they specific and actionable? Replace vague labels ("Submit", "Click here") with clear ones ("Save changes", "Download report").
3. **Visual hierarchy** — Ensure the most important information is the most prominent. De-emphasize secondary content.
4. **Reduce choices** — If there are too many options, group them, hide advanced options, or provide smart defaults.
5. **Flow clarity** — Ensure the user's path through the UI is obvious. Add breadcrumbs, progress indicators, or contextual help where needed.

## Output
- Clarified code with improvements applied
- Before/after summary of changes to labels, hierarchy, and structure
