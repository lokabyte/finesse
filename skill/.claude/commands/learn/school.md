# /school
Start an interactive learning session about the design system — ask questions, get examples, build understanding.

## When to use
- When onboarding to a new design system
- When you need to understand a specific component's API, patterns, or best practices
- When you want to learn the "why" behind design system decisions

## Steps
1. **Identify the topic** — What does the user want to learn? A specific component, a system concept (tokens, density, composition), or general orientation?
2. **Teach with context** — Pull relevant information from the loaded skills: `component-catalog` for components, `color-tokens` for tokens, `density-system` for density, `composition` for compound patterns.
3. **Show, don't just tell** — Provide working code examples for every concept. Show the correct usage alongside common mistakes.
4. **Build incrementally** — Start simple, add complexity. Show the basic usage first, then variants, then composition, then edge cases.
5. **Check understanding** — After explaining, offer a mini challenge: "Can you build X using what we just covered?"

## Output
- Clear explanation of the topic with working code examples
- Common mistakes to avoid
- Related topics to explore next
- Mini challenge for hands-on learning
