# /hush
Reduce the visual intensity of the target — softer colors, lighter weights, more whitespace, subtler interactions. Make it calm.

## When to use
- When a UI feels visually loud or overwhelming
- When too many elements compete for attention
- When a settings page, admin view, or background task needs to recede
- After `/overdrive` went too far

## Steps
1. **Intensity audit** — Identify the loudest elements: bold colors, heavy weights, large sizes, dense information, competing animations.
2. **Color down** — Shift strong colors to their weaker variants (e.g., `bg-brand-default` → `bg-brand-weaker`). Replace solid backgrounds with subtle tints or transparent overlays.
3. **Weight down** — Reduce font weights where possible. Semibold → regular. Bold headings → medium.
4. **Space up** — Increase padding and margins. Let elements breathe. Add whitespace between sections.
5. **Simplify interactions** — Reduce animation durations, make hover effects subtler, soften focus rings.
6. **Mute decorations** — Lighten borders, reduce shadow intensity, remove unnecessary visual separators.

## Output
- Hushed code with reduced intensity
- Summary of what was dialed down
