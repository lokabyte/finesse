# /onboard
Design the onboarding experience for a feature or page — help new users understand what they can do and guide them to their first success.

## When to use
- When launching a new feature that needs user education
- When analytics show users dropping off at a specific step
- When a complex UI needs progressive disclosure

## Steps
1. **Map the journey** — Identify the user's goal, the steps to get there, and where they might get stuck.
2. **Choose the pattern** — Select the right onboarding approach: tooltip tour, inline hints, empty state guidance, progressive disclosure, or interactive walkthrough.
3. **Write the copy** — Craft short, actionable text for each step. Lead with the benefit, not the feature. Use the user's language.
4. **Design the flow** — Sequence the steps logically. Allow skipping. Remember dismissed states. Don't block the UI.
5. **Handle returning users** — Distinguish first-time from returning users. Don't re-show dismissed onboarding. Provide a way to re-access help.

## Output
- Onboarding implementation with components and copy
- Flow diagram showing the sequence and skip paths
- Notes on state management (dismissed, completed, skipped)
